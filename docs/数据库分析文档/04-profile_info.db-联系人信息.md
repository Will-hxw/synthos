# 04 - profile_info.db · 联系人信息

> 存储 QQ 好友列表、好友请求、分组、用户资料等联系人相关数据。

---

## 1. 数据库表一览

| 完成解析 | 表名 | 说明 |
|:---:|:---|:---|
| ✅ | `buddy_list` | 好友相关信息 |
| ✅ | `buddy_req_list_5` | 好友请求/通知 |
| ✅ | `category_list_v2` | 好友分组信息 |
| ✅ | `profile_info_adelie` | 机器人信息 |
| ✅ | `profile_info_v6` | QQ 用户信息（QID、uin、昵称、签名、头像等） |

---

## 2. `buddy_list` — 好友列表

| 列名 | 类型 | 含义 | 说明 |
|:---|:---|:---|:---|
| `1000` | str | `nt_uid` | NT 架构用户唯一标识 |
| `1001` | str | QID | QQ ID（类似微信号） |
| `1002` | int | `uin` | QQ 号 |
| `25007` | int | 分组标识 | 当在默认分组（哪怕已改名）中值为 null |

---

## 3. `buddy_req_list_5` — 好友请求/通知

| 列名 | 类型 | 含义 | 说明 |
|:---|:---|:---|:---|
| `21204` | int | 时间戳 | UTC+8，秒级 |
| `21001` | str | 对方 nt_uid | 请求方的 NT 标识 |
| `20002` | str | 用户昵称 | 请求方的 QQ 昵称 |
| `21502` | int | 申请是否通过 | `0` = 已通过，`1` = 验证中 |
| `21508` | str | 验证消息 | 申请时附带的验证信息 |
| `21509` | str | 申请来源 | 如"通过群聊 xxx" |
| `21505` | int | 验证状态 | `1` = 等待通过，`2` = 同意验证，`13` = 验证过期 |
| `60001` | int | 来源群号 | 如通过群聊发起，记录来源群号 |
| `21501` | int | 申请方标识 | `1` = 他人发送的申请，`0` = 自己发送的申请 |

---

## 4. `category_list_v2` — 好友分组

列 `25011` 为 protobuf 格式，包含分组详情：

| Field Number | 含义 | 说明 |
|:---|:---|:---|
| `25007` | 分组序号 | `0` 为默认分组 |
| `25008` | 分组名称 | 用户自定义的分组名 |
| `25010` | 分组人数 | As uint：该分组总人数 / As sint：当前在线人数 |

---

## 5. `profile_info_v6` — 用户资料信息

存储已接触过的 QQ 用户详细信息（不限于好友）：

| 列名 | 类型 | 含义 | 说明 |
|:---|:---|:---|:---|
| `1000` | str | `nt_uid` | NT 架构用户唯一标识 |
| `1001` | str | QID | 设置了才有，否则为 null |
| `1002` | int | `uin` | QQ 号 |
| `20002` | str | 用户昵称 | 当前 QQ 昵称 |
| `20009` | str | 用户备注 | 用户设置的备注名 |
| `20011` | str | 个性签名 | 当前个性签名 |
| `20004` | str | 头像链接 | URL，需带参数如 `s=640` 或 `s=100` 指定尺寸 |
| `20072` | — | 是否为好友 | hex 值为 `c2e60900` 表示为好友 |
| `21000` | protobuf | "尊贵身份标识符" | 会员/超级会员等身份标识 |
| `24105` | protobuf | 个性签名校验值 | — |

### 5.1 头像链接说明

头像链接为 CDN URL（如 `http://thirdqq.qlogo.cn/...`），通过 `s` 参数控制尺寸：

| s 值 | 尺寸 |
|:---|:---|
| `40` | 40×40 |
| `100` | 100×100 |
| `140` | 140×140 |
| `640` | 640×640（高清） |

---

## 6. `profile_info_adelie` — 机器人信息

存储已交互过的 QQ 机器人（Adelie 平台）的资料信息。具体列名结构与 `profile_info_v6` 类似，字段前缀可能不同。

---

## 7. 数据关联关系

```
buddy_list.1000 (nt_uid)
    └── profile_info_v6.1000 (nt_uid)
        ├── profile_info_v6.1002 (uin)
        ├── profile_info_v6.20002 (昵称)
        └── profile_info_v6.20009 (备注)

buddy_list.25007 (分组标识)
    └── category_list_v2.25007 (分组序号)
        └── category_list_v2.25008 (分组名称)

buddy_req_list_5.21001 (对方 nt_uid)
    └── profile_info_v6.1000 (nt_uid)
```
